The eight CISSP security domains and their approximate exam weightings are:

Security and Risk Management (16%): Focuses on security goals, risk mitigation, compliance, law, regulations, and business continuity.
Asset Security (10%): Covers the collection, handling, storage, retention, and destruction of digital and physical assets.
Security Architecture and Engineering (13%): Includes secure design principles, security models (e.g., Biba, Bell-LaPadula), cryptography, and system vulnerabilities.
Communication and Network Security (13%): Addresses the design and protection of network architecture, secure communication channels, and network components.
Identity and Access Management (IAM) (13%): Focuses on controlling access to systems, user authentication/authorization, and managing the identity lifecycle.
Security Assessment and Testing (12%): Involves designing and performing security testing, audits, vulnerability assessments, and penetration tests.
Security Operations (13%): Covers day-to-day operations such as incident management, disaster recovery, investigations, and resource protection.
Software Development Security (10%): Focuses on applying secure coding practices and security controls within the Software Development Life Cycle (SDLC).
